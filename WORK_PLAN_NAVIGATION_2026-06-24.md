# 2026-06-24 侧边栏导航改造工作计划

## 背景问题

当前左侧侧边栏看起来像多页面导航，但实际点击后大多只是打开抽屉、弹窗或提示，用户会误以为功能失效。

## 本轮目标

- 让侧边栏点击后切换主内容区视图，而不是直接弹出对话框。
- 保留现有数据管理、策略编辑、回测参数设置能力，但改为页面内操作按钮触发。
- 将代码审查相关 Markdown 统一归档，降低项目根目录噪音。

## 已执行

- 归档 `code-review*.md` 和 `code-self-review*.md` 到 `docs/code-reviews/`。
- 新增本工作计划文件，记录本轮导航改造范围。
- 增加 `activeView` 状态，支持数据中心、策略研究、回测中心、因子分析、组合管理、结果分析、报告管理、系统设置等主视图。
- 改造 `Sidebar`，根据当前视图高亮真实菜单项。
- 拆出页面内容组件，让每个菜单都有对应主内容区。
- 保留抽屉/弹窗作为“编辑/管理”的二级动作，不再由侧边栏直接触发。
- 运行后端测试、前端测试、生产构建，并完成浏览器交互验证。
- 增强数据中心主页面，支持在主页面直接同步沪深全 A、选择快照和删除快照。
- 对数据中心主页面新增的同步/选择/删除入口完成测试与浏览器验证。

## 待执行

- 暂无本轮遗留项。

## 后续迭代

- 因子分析、组合管理、报告管理可继续扩展为真实业务页面。
- 未来可引入 React Router，把当前 `activeView` 进一步升级为 URL 路由。
