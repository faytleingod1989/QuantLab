# 2026-06-25 高级策略模板工作计划

## 背景

用户提供的 MindGo 策略包含全市场选股、多条件过滤、按 20 日涨幅排序取前 N、等权买入、固定止损和多出口卖出风控。当前系统原有可视化策略配置器偏简单，难以从零配置这类复合策略。

## 本轮目标

- 新增「主力控盘回踩策略」模板。
- 支持在可视化配置器中一键套用模板，并能看到完整买入/卖出规则链路。
- 扩展回测引擎，让模板中的核心条件真实参与回测。
- 第一版先实现无状态条件、排序 TopN、最大持股数和等权仓位；「生命线跌破后观察 3 天」属于有状态风控，本轮以 MA20 跌破卖出近似实现，后续再增强为观察状态机。

## 2026-06-25 追加完成：生命线观察状态机

- 已新增 `life_line_watch` 规则类型。
- 回测引擎现在会在持仓期间维护每只股票的生命线观察天数：
  - 放量跌破生命线：直接卖出。
  - 普通跌破生命线：直接卖出。
  - 缩量小阴跌破生命线：进入观察。
  - 连续观察达到 3 天仍未站回生命线：卖出。
  - 重新站回生命线：清除观察状态。
- 「主力控盘回踩策略」模板的第 3 个卖出条件已从 MA20 跌破近似规则，替换为真正的「生命线观察」节点。
- 可视化配置器已支持编辑生命线周期、均量周期、观察天数、小阴跌幅下限、放量倍率。
- 新增后端回测测试，验证连续 3 天缩量小阴跌破 MA20 后触发卖出。

## 2026-06-26 追加完成：策略 JSON 导入/导出

- 策略配置弹窗已新增「导出 JSON」能力，导出格式为 QuantLab 可识别的 schema 包装：
  - `schema = quantlab.visual_strategy`
  - `schema_version = 1`
  - `strategy = 当前可视化策略定义`
- 策略配置弹窗已新增两种导入方式：
  - 选择 `.json` 文件导入。
  - 粘贴 JSON 内容导入。
- 导入支持两种 JSON 结构：
  - 纯策略对象。
  - `{ "strategy": {...} }` 包装对象。
- 导入时会进行基础结构校验：
  - 策略名称不能为空。
  - 买入/卖出条件不能为空。
  - 不支持的指标会拒绝导入并提示错误。
  - 已支持的指标会自动补齐默认参数。
- 导入后会替换当前可视化策略，并清空当前 `strategy_version_id`，用户点击「保存新版本」后形成新的策略版本。
