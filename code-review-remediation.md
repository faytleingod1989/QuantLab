# Code Review Remediation

Date: 2026-06-22

本文件记录 `code-review-report.md` 中问题的修复状态，并区分已经闭环的正确性问题与后续架构工作。

## 已完成

| 审查项 | 处理结果 |
|---|---|
| CRIT-01 虚构基准 | 删除与策略收益联动的曲线，改为独立基准行情；离线模式明确标记为“演示沪深300”与 `benchmark.is_demo=true`。 |
| MAJ-01 RSI 非 Wilder 算法 | 改为 SMA 初始值加 Wilder 递推平滑，并添加独立参考测试。 |
| MAJ-02 金额精度 | 价格、现金、成交额及各费用均使用 Decimal 四舍五入到分。 |
| MAJ-03 交易/事件截断 | API 返回完整交易与拒单事件，前端仅在展示层截取最近十笔。 |
| MAJ-04 缺少日志 | 增加任务 ID、开始、完成、拒单、失败日志与不透明错误任务 ID。 |
| MAJ-05 组合资金分配 | 新买入按组合剩余容量、现金及候选信号共同分配。 |
| MAJ-07 固定 CORS | 通过 `QUANTLAB_CORS_ORIGINS` 配置。 |
| 前端固定 API 地址 | 通过 `VITE_API_BASE` 配置。 |
| useEffect 清理 | 首屏请求使用 `AbortController`，避免卸载后更新状态。 |
| 空数据区间 | 返回明确校验错误。 |
| 月收益与夏普 | 月收益使用相邻月末净值，夏普按日收益均值/波动率并年化。 |
| 回测持久化 | SQLite 保存任务、配置快照、进度、取消状态和结果，并支持重启恢复。 |
| CSV 数据集 | 支持 10 MB 限制、OHLC 校验、SHA-256 指纹、去重及预览。 |
| 项目与策略版本 | 增加项目、策略和不可变递增版本；前端支持保存新版本并在刷新后恢复。 |
| 可追溯关联 | 回测记录显式保存 `project_id`、`strategy_id`、`strategy_version_id` 和数据指纹。 |
