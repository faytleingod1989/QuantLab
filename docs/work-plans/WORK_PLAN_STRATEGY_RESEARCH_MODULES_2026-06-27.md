# 策略研究模块优化计划（2026-06-27）

## 背景

当前策略研究已经支持可视化买入/卖出条件组、策略版本、JSON 导入导出和回测联动，但页面信息层级仍偏“规则编辑器视角”。后续要支持更复杂的量化策略，策略研究需要从产品概念上稳定为两层：

1. 选股策略：解决“从哪些股票里挑、怎么筛、怎么排序”。
2. 交易策略：解决“什么时候买、什么时候卖、买多少、如何风控”。

## 推荐方案

采用“两大模块 + 交易内部细分”的结构：

```text
策略研究
├─ 选股策略
│  ├─ 股票池范围：全A / 上证 / 深证 / 创业板 / 科创板 / 自定义
│  ├─ 基础过滤：ST、上市天数、流动性
│  ├─ 候选排序：不排序 / 近期回撤优先 / 强势优先
│  └─ 选股输出：候选股票数量、最多持股数量
│
└─ 交易策略
   ├─ 买入条件组
   ├─ 卖出条件组
   ├─ 仓位管理
   └─ 交易风控
```
